Introduction

Barebones is a boilerplate to help you build your own UI framework. It is not a fully-fledged UI kit or library (Bootstrap, Foundation etc), but a set of tools common to all projects put together in a convenient package to kickstart your next project:

  • a grid system
  • commonly used mixins
  • reponsive helpers

Barebones is design-agnostic and fully customisable, so you can spend less time overwriting and more time creating.

To provide an example, this is the login modal on Airbnb, recreated using the various classes this is the sort of thing you can very quickly create with Barebones.

Why do I need this?

Barebones allows you to quickly build a custom, fully responsive UI. Here's an exmple of how you could use Barebones to replicate the Airbnb login modal.

Prerequisites

Barebones requires SASS v3.3 or later to compile .scss files into .css. Instructions can be found here.

Additionally, vendor prefixing mixins are not used or included as you can use the autoprefixer plugin to do this.

Get started

1. Install the package

Barebones is available as either a Bower or NPM package.

bower install barebones-sass --save
npm install barebones-sass --save-dev

If you want to skip package control and copy the SASS files directly, you can do that too – simply download the latest release from GitHub and integrate the source files directly.

2. Create a settings file

In your project folder, create a _settings.scss file (or whatever you want to call it) with the following values. Each setting is covered in the following section.

$bb-border-radius-scale: 5;

$bb-breakpoints: ();

$bb-colors: ();

$bb-letter-spacing: ();

$bb-line-heights: ();

$bb-global-delimiter: "-";

$bb-helpers: ();

$bb-hover-class-name: "hover";

$bb-max-width-scale: 14;

$bb-responsive: true;

$bb-shadows: ();

$bb-spacing-scale: 10;

$bb-spacing-base: 4px;

$bb-spacing-sides: ();

$bb-text-sizes: ();

Barebones also allows you to specify your own naming convention. For each generated class, you can decide how that class will be outputted. Here's an example: 

$bb-align-items: (
	"modifiers": (
		"flex-start": "items-flex-start",
		"flex-end": "items-flex-end",
		"center": "items-center"
	)
);

// This is used by the align-items helper generator to create classes for the flex align-items property. 
// By default, this will look like this: 

.align-items-flex-start {}
.align-items-flex-start {}
.align-items-flex-start {}

// You can make this less verbose by changing the names: 

$bb-align-items: (
	"modifiers": (
		"flex-start": "items-fs",
		"flex-end": "items-fe",
		"center": "items-c"
	)
);

// Will output: 

.items-fs {}
.items-fe {}
.items-c {}

3. Create your main SCSS file

he final step is to import all these files into your main files that will be compiled into actual CSS. When you're done, it should look something like this:

// Always import the settings file first
@import _settings.scss;

// Always import the settings file first
@import bower-components/barebones-sass/dist/_base.scss;
