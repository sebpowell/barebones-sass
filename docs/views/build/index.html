Getting Started

Barebones is a SASS boilerplate to help you build your own front-end framework. It is not a fully-fledged library, but a set of commonly used tools; a grid system, helper classes and a range of mixins, put together in a convenient package to kickstart your next project. Help you structure your custom SASS framework and provide you with all the tools you need to get up and running.

Everything in Barebones has been stripped back to include only the bare essentials, so you can spend less time overwriting and more time creating something that looks unique. It is a collection of ideas, which we see as 'best practice' collected from around the Internet, and packed together into a single package. We help to help SASS 'newbies' and other front-end.

The best way to use Barebones is to download the source from Github and start building!

Why another framework?

We created Barebones because we weren't happy with other frameworks. Much has been written on the subject of custom SASS frameworks, and we decided to start doing this ourself. As we did, we learned a lot. We wanted it to be as lightweight as possible, whilst also giving enough firection to help you strcuture your SASS project yourself. Buildingin custom CSS frameowrks is hard. 2 years ago, we made the decusiubs of building or own instead of using a full-fledged framework like Bootstrap. There were various reasons for this, but one of the mains reasons was. No vendor-prefixing mixins (these can not be handled using your build process) Many librarties provide incredibly basic mixins that don't actually help you much. For example, the Bourbon library provides mixins to simplify certain commands, but it doesn't actually encourage.

At the same time, we spent a lot of time reading articles by the likes of Hugo Giraduel, Ben Frain and others that regularly had great ideas for structring projects ina more maintable way. Things like managing z-index maps etc. If you're starting out, or for people whgi are used to tradtionally CSS, it takes a long time to build up that knowledgebase. By which time, it may be too late to add in best practices you've seen elsewhere. BArebones is here to give you that leg-up, included a set of 'best practices' which we think have genuinely made our lives easier.

  • the lack of customisation these frameowkr offer. Bootrastp, foundation and other frameworks

How is it different to other frameworks?

  • Written in SASS, the best CSS pre-processor there is
  • Lightweight, includes only the most fundamenttl tools needed in web design (and you can even customise what's included)
  • Fully customisable - it's a boilerplace, not a framework

Installing

You can also get the latest release using Bower.

You can alos simply dowload the source files, include and get started! The issue with this approach though is that it makes it difficult to upgrade. However if you're happy maintaining your own SASS library (which after all, is part of the philshopy behind this project), then you can do so and then simply update the relevant files when the next realse comes out.

Because Barebones doesn't include any components or styling, it won't get in your way. By default, classes are not namespaced, but the project is strcutured to allow you to do this if you wish.

You'll need SASS is not supported by web browser, so you'll need to process. There are various ways of doing this, depending on your steup – see the SASS site for more info

Barebones is split into two sets of files;

  • A collection of the core files
  • A global settings file.

The settings file acts as a global configuration file – mixins, helpers and the grid system all pull i values from this file. You need this file for Barebones to work – it won't work otherwise.

1. Install the Barebones package

bower install barebones

2. Create a settings file (see here for more info)

3. Create a new SASS file, import the settings file first followed by _base.sass.

  • - libs/barebones/_base.scss
  • - assets/scss/styles.scss
  • - assets/scss/_settings.scss

    • Your styles.scss file should look like this:

    @import _settings.scss // Always import the settings file first
@import _base.scss // Import Barebones
// Your stuff can come here.
    @import "_settings.scss"
@import "bower_components/dist/_base"

    Project Structure

    Helpers

    Background Generates background-color helpers for every colour specified in your $colours map.
    Borders Generates border helpers using the $keyline value specified in your $colours map.
    Layout Layout helpers
    Margin & Padding Generates margin and padding helpres from sizes specified in $sizes map.
    Typography Generates text sizing, colours and transform helpers from sizes and colours specified.
    Visibility Generic helpers, alongside generated helpers to target different viewports, using the $breakpoints map.

    Arrow Generates background-color helpers for every colour specified in your $colours map.

    Uppercase Generic helpers, alongside generated helpers to target different viewports, using the $breakpoints map.
    Z-Index Generic helpers, alongside generated helpers to target different viewports, using the $breakpoints map.

    Grid

    Columns Generates background-color helpers for every colour specified in your $colours map.

    Gallery Layout helpers
    Offset Helpers Generates margin and padding helpres from sizes specified in $sizes map.
    Row Generates text sizing, colours and transform helpers from sizes and colours specified.

    Requirements

    Barebones relies heavily on SASS maps to work, so you'll need to make sure you're using SASS 3.3 or higher. Additionally, vendor prefixing mixins are not used or included as you can use the autoprefixer plugin to do this.
