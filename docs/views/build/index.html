Getting Started

Barebones is a SASS boilerplate to help you build a maintanable and scalable front-end framework. It is not a fully-fledged library, but a set of tools common to all projects; a grid system, helper classes and a range of mixins, put together in a convenient package to kickstart your next project. It includes only the bare essentials – so you can spend less time overwriting and more time creating.

Why does the world need Barebones?

It probably doesn't, but here goes; we wanted to create a lightweight SASS library that we could share between projects – powerful enough to allow us to focus on design, and lightweight enough to stay out of our way.

Over time, we've learned a lot about how to structure our SASS projects in a cohesive, scalable and maintainable way. We thought other people might find this useful and decided to release our work in the public domain. My decision was to pick out the best of the techniques I’d discovered and combine them all in one place, so that every new project I work on is using the most modern technologies I know about.

Why build your own framework?

Existing frameworks such as Bootstrap and Foundation are great for prototyping but are typically bloated and lack flexibility (if you want to create your own theme, you're going to spend a lot of time with the !important flag – which is a great way for things to quickly become completley unmanageable).

Buidling your own framework puts you in the driving seat, giving you full control over how your app or website looks, without being shoe-horned into someone else's design system.

comes with a series of SCSS and JS files designed to start all projects at the same point, avoiding the need to rewrite code on every project.The SCSS files reset browser styles to a default point, and includes a large range of reusable mixins. allowing you to quickly get off the ground and begin coding with a grid system, and add other components and modules as required.

Installing Barebones

If you're new to SASS, you'll first need to make sure you're able to compile SASS files.

1. Install the package

Barebones is available as a Bower package. You can also download the latest release from Github.

bower install barebones

2. Create a settings file

Barebones code allows you to define your own settings – just brab the example file and change the values to fit your needs.

3. Create your main SASS file

Import your newly created settings file, followed by _barebones.sass into your main SASS file. It should look like this:

@import _settings.scss // Always import the settings file first
@import bower-components/_base.scss // Import Barebones – the path will depend on your setup.
// Your stuff can come here.

Requirements

Barebones relies heavily on SASS maps to work, so you'll need to make sure you're using SASS 3.3 or higher. Additionally, vendor prefixing mixins are not used or included as you can use the autoprefixer plugin to do this.
